项目开发文档：AI 桌面伴侣 (代号：电车旅行)

1. 项目概述

一款基于 Unity 开发的 Windows 桌面放置类陪伴应用。以《千与千寻》电车场景为视觉核心，通过 AI 接入 Windows 系统进程列表，感知用户当前行为（工作、游戏、休息），提供个性化的情感陪伴、任务拆解与年度总结。

2. 技术栈

引擎: Unity 6.3.9f1 (Target: Windows Standalone)

AI 调用工具:

https://github.com/rageagainstthepixel/com.openai.unity

AI 核心: OpenAI GPT-4o / Gemini 2.5 Flash (用于任务拆解与总结)

底层交互: C# System.Diagnostics (进程监控), Win32 API (窗口置顶/透明度控制)

视觉风格: 2D 手绘风格 (电车、无脸男、窗外流动的风景)

代码架构：采用MVC（Model View Controller）架构

核心规则

View不能直接修改Model的数据

View可以直接读Model的数据/监听事件等等

通过data权限分级来确保View不会越权（reader只能读，writer包含reader+写入权限）

Model最好都通过Zenject的DI来生成和管理dependency，这样一来也方便interface分级和注入。

3. 任务路线图 (按优先级排列)

第一阶段：核心感知与环境 (高优先级 / 核心功能)

最先实现“观察者”身份，这是项目的逻辑基础。

Windows 进程监控模块 (重要性: 极高)

实现 C# 脚本调用 Process.GetProcesses()。

建立“行为映射表”：例如 vscode.exe -> 工作中；steam.exe -> 娱乐中。

隐私处理: 设计本地白名单过滤机制，确保只记录预设程序。

Unity 动态环境搭建 (新颖性: 高)这一块只需要预留接口，美术表现暂时不需要你来实现。

实现窗外风景随真实时间/天气同步（通过外部天气 API）。

根据用户忙碌程度调整“车速”或“光影”。

第二阶段：AI 交互逻辑 (中优先级 / 核心体验)

任务自然语言拆解 (新颖性: 极高)

用户输入一句“人话”，接入 LLM 自动拆分为子任务。

基于表情包的非侵入式对话 (重要性: 高)

最小化干扰设计: 通过角色动作或表情包（Emoji/Sticker）反馈。

第三阶段：情感价值与回顾 (中优先级 / 留存功能)

AI 年度/周度总结 (情感价值: 极高)

将监控数据交由 GPT 处理，生成“陪伴日志”。

自定义提醒系统 (重要性: 中)

4. 技术难点预判

性能占用: 作为背景应用，必须限制 Unity 帧率为 30 或更低，并优化 API 调用频率（如每 5 分钟请求一次行为分析）。

API 密钥安全: 密钥不应硬编码在客户端，建议在本地提供配置文件由用户填写，或通过简单的中转服务器。

5. MVP (最小可行性产品) 目标

一个能显示当前 Windows 活跃软件名的透明窗口。

一个会根据软件名变换表情的无脸男角色。

一个简单的按钮，点击后 AI 会把一句长任务变成短清单。

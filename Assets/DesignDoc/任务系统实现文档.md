# 任务系统实现文档

本文档描述当前项目内「Task 系统 + Quest 系统」的实现状态、数据结构、关键流程与配置方法。

---

## 1. 系统目标与边界

- **Task 系统**：用户可交互的日常待办（增删改拖拽、归档、聊天自动加任务）。
- **Quest 系统**：流程化任务链（配置驱动、条件驱动、通常只读展示在 Task Panel 中）。
- 两者是两套数据与业务逻辑，UI 复用同一面板展示。

---

## 2. Task 系统（现有）

### 2.1 数据模型

- `BigEvent`：大任务（标题、子任务列表、处理状态、错误状态）
  - `Assets/Scripts/Model/TaskDecomposition/BigEvent.cs`
- `SubTask`：小任务（标题、完成状态、顺序）
  - `Assets/Scripts/Model/TaskDecomposition/SubTask.cs`
- 持久化：`tasks.json`
  - `Assets/Scripts/Model/TaskDecomposition/TaskDecompositionModel.cs`

### 2.2 核心控制器

- `TaskDecompositionController`
  - 新增/删除/编辑/拖拽排序
  - 子任务完成切换
  - 大任务归档
  - AI 拆解大任务
  - `Assets/Scripts/Controller/TaskDecompositionController.cs`

### 2.3 UI

- 主界面：`TaskPanelView`
  - 左侧大任务列表，右侧子任务列表
  - 拖拽、内联编辑、完成按钮
  - `Assets/Scripts/View/TaskUI/TaskPanelView.cs`
  - `Assets/UI/TaskPanel.uxml`
  - `Assets/UI/TaskPanel.uss`

## 3. Quest 系统（现有）

### 3.1 配置资产（SO）

- `QuestDatabase`：Quest 列表入口
  - `Assets/Scripts/Core/Settings/QuestDatabase.cs`
  - 资产：`Assets/Data/Quest/QuestDatabase.asset`
- `QuestDefinition`：单个 Quest 定义
  - `questId`、`title`
  - `unlockCondition.requiredQuests`（前置 Quest 列表，AND）
  - `steps`（每步条件）
  - `onCompleteActions`（完成动作列表）
  - `Assets/Scripts/Core/Settings/QuestDefinition.cs`

### 3.2 Step 条件模型

- `QuestStepDefinition`
  - `smartConditionPrompt`（智能条件）
  - `ruleCondition`（规则条件）
  - `conditionLogic`（AND/OR）
  - `checkTiming`（PerChat / PerTaskComplete / Manual）

### 3.3 运行态与持久化

- Runtime：`QuestRuntimeState`
  - 当前激活 quest、每个 quest 的解锁/完成状态、step 完成状态
  - `Assets/Scripts/Model/Quest/QuestRuntimeState.cs`
- 持久化：`quest_runtime.json`
  - `Assets/Scripts/Model/Quest/QuestRuntimeModel.cs`

### 3.4 评估与推进

- `QuestController`
  - 处理 unlock、step 评估、手动完成 quest、触发完成动作、切换 active quest
  - `Assets/Scripts/Controller/QuestController.cs`
- 条件评估链路：
  - `QuestRuleConditionEvaluator`（硬编码规则）
  - `QuestAgentEvaluator`（LLM）
  - `QuestConditionEvaluator`（组合逻辑）
  - `Assets/Scripts/Service/Quest/`

## 4. Quest 在 Task Panel 的交互逻辑

### 4.1 展示方式

- Quest 作为左侧列表中的一个“虚拟大任务”展示。
- 点击后右侧显示 Quest steps（只读样式，方形勾选视觉）。

### 4.2 完成流程（手动收尾）

- 当 Quest 全部 step 完成后，右侧「完成」按钮可点击。
- 点击后：
  1. 触发 `TryCompleteQuest(questId)`
  2. 执行 `onCompleteActions` 列表
  3. 标记当前 quest 完成
  4. 重新计算 unlock 并尝试激活下一条 quest
  5. 刷新列表显示

## 5. OnComplete Actions（数据驱动）

### 5.1 设计

- 基类：`QuestCompleteAction : ScriptableObject`
  - `Assets/Scripts/Core/Settings/QuestActions/QuestCompleteAction.cs`
- Quest 可配置多个动作：`onCompleteActions`
- 执行入口：`QuestActionExecutor`
  - `Assets/Scripts/Service/Quest/QuestActionExecutor.cs`

### 5.2 已实现动作

- `LogQuestCompleteAction`：记录完成日志
- `UnlockEmojiListAction`：解锁某个 EmojiList 到 active palette
- `TriggerProfileAgentAction`：触发 Profile Agent（可配置多个 tier）

对应目录：
- `Assets/Scripts/Core/Settings/QuestActions/`
- `Assets/Data/Quest/Actions/`

---

## 6. Signal 桥接（跨模块解耦）

为避免动作资产直接依赖控制器层，action 通过 signal 触发系统行为：

- `UnlockEmojiListRequestedSignal`
- `TriggerProfileAgentRequestedSignal`
- `Assets/Scripts/Core/Signals/QuestActionSignals.cs`

订阅侧：
- `EmojiChatController`：处理 emoji unlock
- `ProfileController`：处理 profile agent 触发

## 7. 关键配置清单（运行前检查）

1. `ProjectContext.prefab` 已绑定 `QuestDatabase`
   - `Assets/Resources/ProjectContext.prefab`
2. `AgentRegistry.asset` 包含 `quest-agent`
   - `Assets/Data/Quest/QuestAgent.asset`
3. `QuestDatabase.asset` 的 `quests` 必须列出所有要运行的 quest
   - 包括前置和后续 quest（常见漏配点）

## 8. 常见问题排查

### 问题：完成第一条 quest 后没有出现下一条

优先检查：
- `QuestDatabase.asset` 是否包含下一条 quest
- 下一条 quest 的 `requiredQuests` 是否正确引用前置 quest 资产
- `quest_runtime.json` 是否有历史脏状态（可用系统菜单重置）

### 问题：Quest action 执行但效果没变化

优先检查：
- 对应 action 资产字段是否正确配置（例如 `emojiListToUnlock`）
- action 是否加入到 quest 的 `onCompleteActions` 列表
- 相关 signal 是否在 installer 声明
